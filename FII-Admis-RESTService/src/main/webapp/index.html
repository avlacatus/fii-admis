FII Admis REST API {DRAFT}

Github Project: avlacatus/fii-admis

* Input/ Output sunt in formatter json.
* La url-ul http://fii-admis-restservice-tdq5yd2mhw.elasticbeanstalk.com/api se adauga urmatorele uri path-urile.
* ATENTIE! Momentan, nu functioneaza niciun URL!

GET /candidates
- Output: lista de {candidate_id}
- Scop: vezi toti candidatii inscrisi (fara rezultate)

GET /candidates?query

GET /candidates/{candidate_id}
- Output: informatiile de inscriere ale unui candidat: {candidate_id, first_name, last_name, social_id, gpa_grade, a_test_grade}
- Scop: informatiile personale doar unui student

GET /admission_results
- Input: adaugat header "Pragma:admin" (fff secure :)) )
- Output: lista cu {candidate_id, final_grade, status}
- Scop: publicarea rezultatelor (echivalent cu start calcularea admitere)
- Restrictie: prima oara, aceasta operatie este permisa doar de admin, elevu va primi 401 Unauthorized; apoi va fi permisa si din interfata de elevi

GET /admission_results?query (media finala, nume)

GET /admission_results/{candidate_id}
- Output: informatiile unui cantitat despre rezultatul sau: {candidate_id, final_grade, status}
- Scop: informatii de admitere doar pentru un student
- Restrictie: aceasta operatie va fi permisa doar dupa ce un admin va data GET la /admission_results

- - - - - - - - - - - - - - - -

POST /candidates
- Input: un json cu datele {first_name, last_name, social_id, gpa_grade, a_test_grade}
- Output: 201 Created + Header: "Location: /{candidate_id}"
- Scop: se inregistreaza elevul la facultate

- - - - - - - - - - - - - - - -

PUT /candidates/{candidate_id}
- Input: un json cu datele {candidate_id, first_name, last_name, social_id, gpa_grade, a_test_grade}
- Output: 204 No Content
- Scop: elevul sa-si updateze datele personale

- - - - - - - - - - - - - - - -

DELETE /candidates/{candidate_id}
- Input: no content
- Output: 204 No Content
- Scop: elevul vrea sa renunte la admitere
